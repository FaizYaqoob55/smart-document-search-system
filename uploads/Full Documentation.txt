TEAMFLUX – COMPLETE PROJECT DOCUMENTATION
:pushpin: Overview
TeamFlux is a backend-based team and project management system designed to manage teams, projects, growth sessions, technical debt, and workflows efficiently. It provides secure access using Role-Based Access Control (RBAC) and enables structured collaboration within teams.
:dart: Objectives
Manage teams and projects efficiently
Track employee growth sessions and performance
Handle technical debt and system improvements
Provide secure collaboration between team members
Deliver structured and scalable backend APIs
:hammer_and_wrench: Tech Stack
FastAPI
PostgreSQL
SQLAlchemy
JWT Authentication
Redis (optional for caching)
:gear: COMPLETE SYSTEM FLOW (VERY IMPORTANT)
:repeat: End-to-End Flow
User registers/logs in (JWT token is generated)
User is assigned to a team
Each team contains multiple projects
Inside each project:
Growth Sessions are conducted
Notes are added
Action Items are assigned
Technical Debt is tracked
Comments and Mentions are used for collaboration
Deprecation plans are managed
Dashboard displays overall summary

:rocket: SETUP GUIDE (IMPORTANT)
1. Clone Repository
Bash
git clone <repo-link>
cd teamflux
2. Install Dependencies
Bash
pip install -r requirements.txt
3. Setup Database
Install PostgreSQL and create database:
SQL
CREATE DATABASE teamflux;
4. Run Migrations
Bash
alembic upgrade head
5. Run Server
Bash
uvicorn app.main:app --reload
6. Open Swagger Docs
Plain text
http://localhost:8000/docs
