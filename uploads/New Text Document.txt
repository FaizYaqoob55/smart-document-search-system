An API (Application Programming Interface) is a set of rules that allows different software applications to communicate and share data with one another. It acts as a middleman, translating a request from one system (the client) and delivering a response from another (the server)

How an API Works (The Restaurant Analogy)
A common way to understand APIs is the restaurant analogy: 
Client (You): The customer sitting at a table who wants to order food.
Server (The Kitchen): The place where the food is prepared.
API (The Waiter): The intermediary who takes your order, delivers it to the kitchen, and brings the food back to you. You don't need to know how the chef cooks; you just need to know how to talk to the waiter.

Core Components
Endpoints: Specific URLs or "addresses" where the API receives requests.
Methods: Standard instructions that define the action (e.g., GET to retrieve data, POST to create new data, DELETE to remove it).
Request/Response: The formatted message sent by the client and the structured data (often in JSON or XML) sent back by the server.
